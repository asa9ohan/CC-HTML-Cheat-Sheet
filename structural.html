Structural Tags

The <main> and <footer> structural elements, along with <nav> and <header> help describe where an element is located based on conventional web development standards.


Opening Tag Closing Tag Description
<main> </main> Used to encapsulate the dominant content within a webpage, separate from the <footer> and <nav> sections of a webpage.
<footer> </footer> Contains information such as contact information, copyright information, terms of use, site map, references to top of page links.
<section> </section> Defines elements in a document, such as chapters, headings, or any other areas of the document with the same theme.
<article> </article> Defines an article section to hold articles, blogs, comments, magazines, etc. Might contain a combination of text, images, audio, etc.
<aside> </aside> Used to mark additional information that can enhance another element but isn't required in order to understand the main content. Can be used alongside elements such as <article> or <section>.

Common uses include bibliographies, endnotes, comments, pull quotes, editorial sidebars, additional information.
<figure> </figure> Used to encapsulate media such as an image, illustration, diagram, code snippet, etc., which is referenced in the main flow of the document.
<figcaption> </figcaption> Used to describe the media in the <figure> tag, usually inserted inside the <figure> tags.
