Media Tags

Media tags have various attributes that affect playback.


Opening Tag Closing Tag Description Attributes Description
<audio> </audio> Used to embed audio content into a document. src="audio.mp3" src attribute, used to link the audio source.
type="audio/mp3" audio attribute, used to name what kind of audio (not always necessary, but recommended: can help browser identify if file type is supported by browser)
controls controls attribute, automatically displays audio controls into the browser, such as play and mute
autoplay autoplay attribute, adds autoplay functionality
Other useful <audio> attributes: MDN Docs
<source> (self-closing) Used to indicate the source of a media file separately from (inside) the media (<audio>, <img>, etc.) tag.

🚩 Why use a separate tag?

The <audio> tag can be used in the same way as the <video> tag:

<audio
  controls
  src="/media/examples/t-rex-roar.mp3">Your browser does not support the <code>audio</code> element.
</audio>

However, not all browsers support the same file types, so you might want to use multiple sources as backup:

<audio controls>
  <source src="myAudio.mp3" type="audio/mpeg">
  <source src="myAudio.ogg" type="audio/ogg">
  <p>Your browser doesn't support HTML5 audio. Here is a <a href="myAudio.mp4">link to the audio</a> instead.</p>
</audio>
src="myAudio.mp3"
src="myAudio.ogg"		 Indicate media source.
type="audio/mpeg"
type="audio/ogg"		 Indicate file type.
